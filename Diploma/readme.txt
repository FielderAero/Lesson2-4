1) На странице main.php Находится таблица со списком тем для пользователя, и вход через сессии для администратора. 

2) После входа администратора попадаем на страницу entered.php

Данные по администраторам храняться в базе u22733_vergilon, в таблице adminslistспис
Через эту страницу можем попасть на список администраторов, где можем совершать какие либо действия.

Можем на список тем, где можем видеть сколько у нас вопросов без ответа, опубликовано и т.д.

Можем на требуемый в конце задания список с вопросами на которые еще не даны ответы независимо от темы, сортированные по дате.

3) Страница со списком админов это adminlist.php В ней в принципе ничего интересного и сложного не было.

4) Страница со списком тем это themeslist. Для тем у меня создана отдельная база данных u22733_netology

Количество вопросов и количество вопросов без ответов определяется через тот же цикл, через который строится таблица со списком тем. Цикл проходит
по каждой теме и смотрит сколько у нас есть непустых колонок question, и сколько же пустых колонок answer. 

Забегая немного вперед скажу, что число опубликованных вопросов у меня определяется через отдельную колонку в таблице. Если значение 1 то опубликовано, если 0 то скрыто. 
По умолчанию в колонке published стоит 0.

Ну и удалить, там все просто. По нажатии кнопки удаляет тему с определенным именем. Там можем совершать всякие операции с вопросами

Да и поначалу может ввести в заблуждение строка $pdo2 = new PDO("mysql:host=localhost; dbname=u22733_vergilon; charset=UTF8;", "u22733_vergilon", "qwerty12345"); 

Она введена для того чтобы у нас правильно работал один из последних пунктов. Там где "в конце задания список с вопросами на которые еще не даны ответы независимо от темы, сортированные по дате".

5) С этой страницы можем попасть на список вопросов theme.php который генерируется в зависимости от выбранной темы. Там можем совершать всякие операции с вопросами.

6) Так, ну вот самый сложный для меня пункт. Где нужно вывести список вопросов из всех тем по дате. Реализовал я это так. Страница questionswithnoanswers.php

Сначала запустил цикл на загрузку списка тем(таблиц) с вопросами и внутрь этого цикл засунул еще один, который пишет вопросы без ответа из всех тем, в одну общую таблицу,
которая лежит в другой базе данных. В той же где список администраторов. u22733_vergilon.

А так как мне нужно, чтобы при редактировании этих вопросов, тем же образом изменялись и их оригиналы в первичных таблицах, то затем мне и надо частенько
обращаться одновременно к двум базам данных. Вот таким образом если я редактирую вопрос в этой общей таблице со списком вопросов по дате, то у меня редактирование происходит
и в оригинальной файле. Наоборот это тоже работает. 

Поэтому у меня много где непонятная на первый взгляд строка $pdo2 = new PDO("mysql:host=localhost; dbname=u22733_vergilon; charset=UTF8;", "u22733_vergilon", "qwerty12345");
